# MMD to Funscript Converter (Ultimate Edition v4.1)

Blender에서 MMD 모델의 뼈(Bone) 애니메이션을 가상 장치 제어용 포맷인 **Funscript** 파일로 변환해 주는 애드온입니다. 본 가이드는 애드온 UI 패널 내부의 각 기능과 옵션에 대해 상세히 설명합니다.

---

## 🖥️ UI 기능 상세 설명

Blender의 `3D Viewport > Sidebar (N-Panel) > MMD Funscript` 탭에서 가동되는 패널의 상세 구성입니다.

### 1. 대상 모델 및 뼈 지정 (Model & Bone Selection)

변환의 중심이 될 모델과 추적할 본(Bone)을 선택합니다.

* **모델 (Model Display)**: 현재 뷰레이어에서 활성화된 아머처(Armature) 객체의 이름을 표시합니다. 아머처가 선택되지 않으면 경고가 표시됩니다.
* **뼈 지정 방식 (Bone Selection Mode)**: 세 가지 탐색 모드를 지원합니다.
* `1. 자동 감지 (AUTO)`: 스크립트 내부의 우선순위 목록(`하반신 ➔ 허리 ➔ 센터`) 순서대로 아머처 내부에서 일치하는 본을 자동 검색합니다.
* `2. 추천 뼈 목록 필터링 (FILTERED)`: 아머처 내부의 본 중 데이터 추출에 최적화된 추천 본 목록만 드롭다운에 노출합니다.
* `3. 모든 뼈 목록 중 선택 (ALL)`: 아머처 내부에 존재하는 모든 본 목록 중에서 사용자가 자유롭게 대상을 지정합니다.


* **대상 뼈 (Selected Bone)**: 자동 감지 모드가 아닐 때, 직접 드롭다운이나 검색을 통해 변환할 대상 본을 입력하는 칸입니다.

### 2. 출력 및 환경 설정 (General Settings)

* **저장 경로 (target_dir)**: 생성된 `.funscript` 파일이 저장될 로컬 디렉토리 경로를 지정합니다.
* **파일 이름 (file_name)**: 생성될 스크립트의 기본 파일명을 입력합니다 (기본값: `motion`).
* **프레임 간격 (frame_step)**: 애니메이션을 몇 프레임 주기로 샘플링하여 기록할지 지정합니다 (1~30 프레임 범위, 기본값: 5). 값이 작을수록 정밀하고 촘촘한 스크립트가 생성됩니다.
* **추출 방식 (Tracking Mode)**: 회전각 연산의 기준점을 설정합니다.
* `세계/화면 기준 (Global)`: 3D 공간의 절대 격자 및 화면 고정 시점을 기준으로 연산합니다.
* `캐릭터 기준 (Local)`: 캐릭터 자체의 몸 방향을 기준으로 앞뒤, 좌우, 회전 움직임을 계산합니다.



### 3. 글로벌 옵션 & 가동 범위 스캔 (Global & Range Options)

* **경계 지연 추적 사용 (use_smoothing)**: 가동 범위를 일시적으로 벗어날 때 급격하게 튀는 현상을 방지하는 보간 기능입니다.
* `지연 추적 프레임`: 지연 추적 시 보간을 적용할 프레임 단계를 조절합니다.


* **자동 범위 최적화 (Auto-Range)**: 애니메이션 전체를 미리 분석하여 움직임의 최소/최대 한계치를 기준으로 스크립트 범위를 자동으로 100% 최적화해 매핑합니다.
* **전체 프레임 정밀 스캔 (use_force_pre_scan)**: 프레임 간격을 건너뛰지 않고 전체 프레임을 정밀하게 1차 스캔하여 뒤돌기 경계면 보정의 정확도를 극대화합니다.
* **범위 지정 방식 (Range Mode)**: `자동 범위 최적화`가 비활성화되어 있을 때 수동으로 범위를 지정하는 방식입니다.
* `전체 범위 크기 (Width)`: 설정한 중심 기준점으로부터 지정한 총 각도 너비를 균등 분할합니다.
* `최소/최대 직접 지정`: 최소 각도와 최대 각도를 수치로 개별 제어합니다.



### 4. 축별 제어 옵션 (Rotation Axes: Pitch, Roll, Twist)

X축(Pitch), Y축(Roll), Z축(Twist)의 회전 데이터를 개별적으로 제어하는 독립된 제어 박스들입니다.

* **활성화 체크박스**: 해당 회전축의 독립적인 `.funscript` 파일 추출 여부를 결정합니다.
* **방향 전환**: 활성화 시 출력 데이터의 고점(100)과 저점(0) 방향을 반대로 뒤집습니다.
* **90도 반전**: 회전각이 90도를 초과할 때 이를 다시 반대 방향(Ping-Pong)으로 꺾어 가동 범위(0~100) 내에 안착시키는 옵션입니다.
* **뒤돌기 보정 (Shift-Back)**: 캐릭터가 180도 경계면을 넘어가거나 0도 근방으로 복귀할 때 값이 급격하게 튀는 스냅 현상을 양방향으로 정밀 보정합니다.
* `임계값 (Flip Threshold)`: 뒤돌기 판단 기준이 되는 프레임 간 순간 각도 변동 한계치입니다.


* **수동 범위 설정**: 범위 지정 방식에 따라 `전체 범위(도)` 혹은 `최소/최대` 한계 각도를 직접 입력할 수 있습니다.

### 5. 스트로크 제어 옵션 (Stroke Box)

v4.1에서 강화된 본의 상하(Z축) 이동 거리를 추적하는 특화 기능 섹션입니다.

* **Stroke 활성화**: 회전이 아닌 본의 상하 이동 피치 데이터를 추출하여 메인 `.funscript` 파일로 내보냅니다.
* **방향 전환**: 스트로크의 업/다운 방향을 반전합니다.
* **기본 가동 막대 길이**: 상하 이동 거리를 0~100% 매핑으로 변환할 기준 길이 단위를 의미합니다. (v4.1 수정으로 사용자가 임의 변경한 수치가 강제 리셋되지 않고 유지됩니다.)
* **가동 막대 동적 변형 사용**: 본이 설정된 가동 범위를 벗어날 경우 실시간으로 해당 시점의 양 극점을 구하여 임시 범위 변형을 일으켜 끊김 없는 유연한 매핑을 지원합니다.
* **기립 판별 사용 (Standing Detection)**: 하반신 본의 전체적인 높낮이 변화를 실시간으로 추적하여 상태 전환식 비선형 매핑을 처리합니다.
* `극점 판단 유예 길이 (Grace Count)`: 최저점 및 최고점을 보다 정밀하게 탐색하기 위해 정점 감지 시 추가로 체크할 프레임 단계 수입니다.
* `대퇴골 뼈 지정`: 판단 막대 길이 조정을 위한 대퇴골(Thigh) 본을 지정합니다.
* `판단 막대 길이`: 기립 판별 상태 전환의 기준이 되는 가상의 판단 영역 스케일 길이입니다.

## 💾 실행 버튼 (Execution)

UI 패널 하단의 레이아웃에는 두 개의 실행 버튼이 나란히 배치되어 있습니다.

1. **Funscript 파일 생성 (Normal)**:
* 설정한 옵션에 따라 정제된 최종 `.funscript` 파일들을 지정 경로에 추출합니다.
* 각 활성화된 축에 맞춰 `.pitch.funscript`, `.roll.funscript`, `.twist.funscript` 형태 및 메인 stroke 파일이 생성됩니다.


2. **콘솔 모양 아이콘 버튼 (Debug Mode)**:
* 내부 연산 로직과 매핑 과정을 역추적할 수 있는 디버그 모드 전용 파일(`{파일이름}.debug.funscript`)을 생성합니다.
* 이 파일에는 매 프레임의 스트로크 변형 상태, 바(Bar) 영역의 유동적 변화 및 적용 옵션 메타데이터가 구조화된 JSON 로그 형태로 담겨 있어 연산 트러블슈팅에 유용합니다.
