# mmd_to_funscript (Smart Dynamic v4.7)
Blender Addon

blender_mmd_tools  
https://github.com/MMD-Blender/blender_mmd_tools  

블렌더에 추가하여 MMD 모델 및 일반 아머처(Armature)의 뼈(Bone) 거동 데이터를 분석하고, 가상 장치 제어용 포맷인 고정밀 Funscript 파일로 추출합니다.

## 🚀 v4.7 핵심 업데이트 내역 (v4.1 ➔ v4.7)

1. **스마트 프레임 최적화 압축 (RDP 알고리즘) 도입**
   * 프레임 간격을 `1`로 두어 전수 조사를 진행하더라도, 움직임이 거의 없는 구간이나 직선 궤적의 불필요한 키프레임은 오차 범위 내에서 자동으로 병합·생략됩니다. **용량은 수십 배 압축되면서 모션의 정밀도는 완벽히 보존**됩니다.
2. **축별 상대 허용 오차(Tolerance) 분리 제어**
   * 회전축과 스트로크축의 연산 특성을 고려하여, 압축 오차 범위를 `회전 상대 허용 오차`와 `Stroke 상대 허용 오차`로 분리하여 디테일을 개별 제어할 수 있습니다.
3. **무제한 역방향 극점 탐색 및 동적 검증 알고리즘**
   * 기립/좌식 판별 시 노이즈로 인해 모션이 튀는 현상을 막기 위해, 캐시 기반의 **무제한 역방향 극점 탐색 루프**와 **'판단 유예 오차 범위(%)'** 메커니즘을 적용하여 비선형 매핑 안정성을 극대화했습니다.
4. **원클릭 기본값 초기화(Reset) 오퍼레이터 추가**
   * 대상 아머처의 신체 스케일을 실시간으로 스캔하여 가동 막대 및 판단 막대 길이를 캐릭터 맞춤형 기본값으로 즉시 세팅해주는 초기화 버튼이 추가되었습니다.

## 🖥️ UI 기능 및 파라미터 상세 설명

Blender의 `3D Viewport > Sidebar (N-Panel) > MMD Funscript` 탭에서 사용할 수 있는 기능 리스트입니다.

### 1. 대상 모델 지정 및 초기화 (Model & Smart Reset)
* **모델 (Model Display)**: 현재 선택되어 활성화된 아머처(Armature) 객체의 이름을 표시합니다.
* **🔄 기본값 초기화 (Reset)** `[신규]`: 클릭 시 현재 아머처의 실제 하반신 및 대퇴골(Thigh) 뼈 길이를 분석하여, 플러그인 내 모든 막대 길이 파라미터를 해당 캐릭터 스케일에 가장 이상적인 수치로 자동 리셋합니다.

### 2. 대상 뼈 지정 (Bone Selection)
* **뼈 지정 방식 (Bone Selection Mode)**:
  * `1. 자동 감지 (AUTO)`: 우선순위 기준(`하반신 ➔ 허리 ➔ 센터`)에 맞춰 아머처 내부에서 본을 자동 검색합니다.
  * `2. 추천 뼈 목록 필터링 (FILTERED)`: 주요 핵심 본 목록만 필터링하여 드롭다운에 노출합니다.
  * `3. 모든 뼈 목록 중 선택 (ALL)`: 아머처 내의 모든 본 목록을 드롭다운에 노출합니다.
* **대상 뼈 (Selected Bone)**: 수동 모드일 때, 데이터를 추출할 대상 본을 지정하는 칸입니다.

### 3. 출력 및 환경 설정 (General Settings)
* **저장 경로 (target_dir)**: 생성된 `.funscript` 파일이 저장될 로컬 폴더 경로입니다.
* **파일 이름 (file_name)**: 생성될 스크립트 파일의 이름입니다 (기본값: `motion`).
* **정밀 탐색 프레임 간격 (frame_step)**: 애니메이션 샘플링 주기를 설정합니다. v4.7은 강력한 압축 엔진이 내장되어 있으므로 **`1` 설정을 적극 권장**합니다.

### 4. 스마트 최적화 & 압축 옵션 (Smart Compression)
* **스마트 프레임 최적화 압축**: RDP 알고리즘 활성화 여부를 결정합니다. (체크 권장)
* **회전 상대 허용 오차 (%)**: X, Y, Z 회전축 데이터의 압축 민감도입니다. (기본값: `2.0%`)
* **Stroke 상대 허용 오차 (%)** `[신규]`: Z축 상하 스트로크 데이터의 압축 민감도입니다. 이 값을 조절하여 스트로크의 용량을 획기적으로 줄일 수 있습니다. (기본값: `4.0%`)

### 5. 가동 범위 판단 및 추출 방식 (Tracking & Range)
* **추출 방식 (Tracking Mode)**:
  * `세계/화면 기준 (Global)`: 3D 공간의 절대 공간 격자를 기준으로 연산합니다.
  * `캐릭터 기준 (Local)`: 캐릭터가 바라보는 몸 방향을 회전 기준으로 연산합니다.
* **범위 지정 방식 (Range Mode)**: 범위를 지정하는 방식입니다.
  * `전체 범위 크기 (Width)`: 설정한 중심 기준점으로부터 지정한 총 각도 너비를 균등 분할합니다.
  * `최소/최대 직접 지정`: 최소 각도와 최대 각도를 수치로 개별 제어합니다.

### 6. 축별 회전 제어 옵션 (Pitch, Roll, Twist Box)
X축(Pitch), Y축(Roll), Z축(Twist)의 회전 데이터를 개별적으로 제어합니다.
* **활성화 체크박스**: 해당 회전축의 독립적인 `.funscript` 파일 추출 여부를 선택합니다.
* **방향 전환**: 출력 데이터의 고점(100)과 저점(0) 방향을 뒤집습니다.
* **90도 반전**: 회전각이 90도를 초과할 때 이를 반대 방향(Ping-Pong)으로 꺾어 가동 범위 내에 안착시킵니다.
* **뒤돌기 보정 (Shift-Back)**: 180도 경계면을 넘어가거나 0도 복귀 시 순간적으로 값이 튀는 현상을 정밀 보정합니다.
  * `임계값 (Flip Threshold)`: 뒤돌기 판단 기준이 되는 프레임 간 순간 각도 변동 한계치입니다.
* **범위 지정 (Range)**: 각도 한계를 제어합니다.

### 7. 스트로크 제어 옵션 (Stroke Box)
본의 상하(Z축) 이동 거리를 추적하여 메인 스트로크 데이터를 추출하는 특화 기능 섹션입니다.
* **Stroke 활성화**: 본의 상하 이동 데이터를 메인 `.funscript` 파일로 내보냅니다.
* **방향 전환**: 스트로크의 업/다운 방향을 반전합니다.
* **기본 가동 막대 길이 / 배율 (Multiplier)** `[신규]`: 상하 이동 거리를 0~100% 매핑으로 변환할 기준 길이 및 가중치 배율입니다.
* **가동 막대 동적 변형 사용**: 본이 가동 범위를 벗어날 경우 실시간으로 극점을 재계산하여 끊김 없는 유연한 매핑을 지원합니다.
* **기립 판별 사용 (Standing Detection)**: 하반신 높낮이 변화를 실시간 추적하여 상태 전환식 비선형 매핑을 처리합니다.
  * `극점 판단 유예 길이 (Grace Count)`: 정점 감지 시 노이즈 판단을 유예하고 추가로 탐색할 프레임 수입니다.
  * `판단 유예 오차 범위 (%)` `[신규]`: 미세한 진동 노이즈로 인해 기립 상태가 오판되는 것을 방지하는 허용 편차입니다.
  * `대퇴골 뼈 지정`: 기립 여부를 정밀 대조하기 위한 대퇴골(Thigh) 본 이름입니다. (자동 필터링 추천 지원)
  * `판단 막대 길이`: 기립 판별 상태 전환의 기준이 되는 영역의 스케일 길이입니다.

## 💾 실행 및 디버그 (Execution)

UI 패널 최하단에는 두 개의 버튼이 위치해 있습니다.

1. **Funscript 파일 생성 (Export)**:
   * 설정한 옵션과 RDP 압축 알고리즘에 맞춰 고밀도/저용량으로 정제된 최종 `.funscript` 파일들을 지정 경로에 추출합니다.
   * 각 축 활성화에 따라 `.pitch.funscript`, `.roll.funscript`, `.twist.funscript` 및 메인 스트로크 파일이 컴팩트 포맷(JSON 공백 제거)으로 생성됩니다.
2. **콘솔 모양 아이콘 버튼 (Debug Export)**:
   * 압축 전의 원본 연산 로직, 매 프레임의 스트로크 변형 상태 및 매핑 메타데이터가 가독성 있게 정렬된 가이드용 디버그 파일(`{파일이름}.debug.funscript`)을 생성합니다. 데이터 분석 및 트러블슈팅 시 유용합니다.
