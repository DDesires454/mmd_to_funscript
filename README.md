# mmd_to_funscript
Blender Addon — **v5.3.0**

blender_mmd_tools
https://github.com/MMD-Blender/blender_mmd_tools

블렌더에 추가하여 MMD 모델 및 일반 아머처(Armature)의 뼈(Bone) 거동 데이터를 분석하고, 가상 장치 제어용 포맷인 고정밀 Funscript 파일로 추출합니다.

## 🆕 v5.3.0 업데이트 요약

* **Twist(비틀림) 연산 완전 재작성 — 본의 길이 방향 기준 보정**: 기존에는 카메라/월드
  기준의 고정된 축으로 Twist를 계산해, 캐릭터가 눕거나 크게 회전하면 값이 부정확하거나
  불연속적으로 튀는 경우가 있었습니다. v5.3.0부터는 본의 길이 방향(로컬 Y축) 자체를
  기준으로 삼아 계산합니다.
  * **LOCAL 모드**: 쿼터니언 기반 **Swing-Twist Decomposition**을 도입해, 본의 로컬
    Y축을 회전축으로 두고 다른 축의 휘어짐(Swing) 성분을 제거한 순수 Twist 성분만
    고립해 추출합니다.
  * **GLOBAL 모드**: 카메라 기준 우측 벡터를 본의 현재 길이 방향에 수직인 평면 위로
    매 프레임 동적으로 투영(Dynamic Projection Reference Frame)해, 캐릭터가 눕거나
    회전해도 본의 길이 방향과 일관되게 정렬된 기준축으로 Twist를 계산합니다. 투영이
    거의 실패하는 특이 각도(투영 벡터 길이가 0에 가까운 경우)에는 화면 위쪽 축을
    대신 투영해 회피합니다.
* **중점 반동 인터셉트 감지 시점 보완**: 회전(Pitch/Roll) 동적 변형의 선행(역행 전)
  스캔 도중에도 값이 중점(midpoint)을 넘어서는 순간을 즉시 감지하도록 수정되었습니다.
  기존에는 이 감지가 스캔이 끝난 뒤에만 이뤄져, 특정 구간에서 보정이 한 프레임 늦게
  걸리는 경우가 있었습니다. (ROT_UP/ROT_DOWN 두 스캔 방향 모두 적용)
* 사용자가 직접 조정하는 옵션 항목(프로퍼티) 자체는 v5.2.0과 동일합니다 — 이번 버전은
  연산 로직 정확도 개선이 중심이며, UI/CLI에 새로 추가된 항목은 없습니다.

## 🖥️ UI 기능 및 파라미터 상세 설명

Blender의 `3D Viewport > Sidebar (N-Panel) > MMD Funscript` 탭에서 사용할 수 있는 기능 리스트입니다.

### 1. 대상 모델 지정 및 초기화 (Model & Smart Reset)
* **모델 (Model Display)**: 현재 선택되어 활성화된 아머처(Armature) 객체의 이름을 표시합니다.
* **🔄 기본값 초기화 (Reset)**: 클릭 시 현재 아머처의 실제 하반신 및 대퇴골(Thigh) 뼈 길이를 분석하여, 플러그인 내 모든 막대 길이 파라미터를 해당 캐릭터 스케일에 가장 이상적인 수치로 자동 리셋합니다. 카메라 투영 설정(바라보는 방향 `-Y` / 화면 위쪽 `+Z`)과 Pitch·Roll 동적 변형(꺼짐) 값도 함께 초기화됩니다.

### 2. 대상 뼈 지정 (Bone Selection)
* **뼈 지정 방식 (Bone Selection Mode)**:
  * `1. 자동 감지 (AUTO)`: 우선순위 기준(`하반신 ➔ 허리 ➔ 센터`)에 맞춰 아머처 내부에서 본을 자동 검색합니다.
  * `2. 추천 뼈 목록 필터링 (FILTERED)`: 주요 핵심 본 목록만 필터링하여 드롭다운에 노출합니다.
  * `3. 모든 뼈 목록 중 선택 (ALL)`: 아머처 내의 모든 본 목록을 드롭다운에 노출합니다.
* **대상 뼈 (Selected Bone)**: 수동 모드일 때, 데이터를 추출할 대상 본을 지정하는 칸입니다.

### 3. 출력 및 환경 설정 (General Settings)
* **저장 경로 (target_dir)**: 생성된 `.funscript` 파일이 저장될 로컬 폴더 경로입니다.
* **파일 이름 (file_name)**: 생성될 스크립트 파일의 이름입니다 (기본값: `motion`).
* **정밀 탐색 프레임 간격 (frame_step)**: 애니메이션 샘플링 주기를 설정합니다. 강력한 압축 엔진이 내장되어 있으므로 **`1` 설정을 적극 권장**합니다.

### 4. 스마트 최적화 & 압축 옵션 (Smart Compression)
* **스마트 프레임 최적화 압축**: RDP 알고리즘 활성화 여부를 결정합니다. (체크 권장)
* **회전 상대 허용 오차 (%)**: X, Y, Z 회전축 데이터의 압축 민감도입니다. (기본값: `2.0%`)
* **Stroke 상대 허용 오차 (%)**: Z축(또는 카메라 Up 기준) 상하 스트로크 데이터의 압축 민감도입니다. 이 값을 조절하여 스트로크의 용량을 획기적으로 줄일 수 있습니다. (기본값: `4.0%`)

### 5. 가동 범위 판단 및 추출 방식 (Tracking & Range)
* **추출 방식 (Tracking Mode)**:
  * `세계/화면 기준 (Global)`: 3D 공간의 절대 공간 격자를 기준으로 연산합니다. 이 모드를 선택하면 아래 **글로벌 투영 가상 카메라 설정**이 함께 나타납니다.
  * `캐릭터 기준 (Local)`: 캐릭터가 바라보는 몸 방향을 회전 기준으로 연산합니다.
* **글로벌 투영 가상 카메라 설정** `[v5.0.0 신규, Global 모드 전용]`:
  * `바라보는 방향 (Look From)`: 어느 축 방향에서 모델을 바라볼지 결정합니다 (+X/-X/+Y/-Y/+Z/-Z 중 선택, 기본값 `-Y`, 정면).
  * `화면 위쪽 방향 (Up Axis)`: 화면의 위쪽이 될 세계 기준 축을 결정합니다 (기본값 `+Z`). 이 축의 투영 값을 기준으로 Stroke와 기립 판단이 동작합니다.
  * ⚠️ 두 값을 동일한 축으로 지정하면 UI에 경고가 표시되며, 실제 연산 시에는 안전한 기본 정면 뷰(`-Y`/`+Z`)로 자동 대체됩니다.
* **범위 지정 방식 (Range Mode)**: 범위를 지정하는 방식입니다.
  * `전체 범위 크기 (Width)`: 설정한 중심 기준점으로부터 지정한 총 각도 너비를 균등 분할합니다.
  * `최소/최대 직접 지정`: 최소 각도와 최대 각도를 수치로 개별 제어합니다.

### 6. 축별 회전 제어 옵션 (Pitch, Roll, Twist Box)
X축(Pitch), Y축(Roll), Z축(Twist)의 회전 데이터를 개별적으로 제어합니다.
* **활성화 체크박스**: 해당 회전축의 독립적인 `.funscript` 파일 추출 여부를 선택합니다.
* **Pitch / Roll 동적 변형 사용** `[v5.0.0 신규]`: 값이 지정 범위를 벗어나면 Stroke처럼 가동 막대의 극점을 실시간으로 새로 구하고, 값이 초기 중앙점을 다시 통과하는 순간을 감지해 자연스럽게 복귀시키는 "중점 반동 인터셉트" 보정을 적용합니다. (Twist에는 적용되지 않습니다.)
* **방향 전환**: 출력 데이터의 고점(100)과 저점(0) 방향을 뒤집습니다.
* **90도 반전**: 회전각이 90도를 초과할 때 이를 반대 방향(Ping-Pong)으로 꺾어 가동 범위 내에 안착시킵니다.
* **뒤돌기 보정 (Shift-Back)**: 180도 경계면을 넘어가거나 0도 복귀 시 순간적으로 값이 튀는 현상을 정밀 보정합니다.
  * `임계값 (Flip Threshold)`: 뒤돌기 판단 기준이 되는 프레임 간 순간 각도 변동 한계치입니다.
* **범위 지정 (Range)**: 각도 한계를 제어합니다.

### 7. 스트로크 제어 옵션 (Stroke Box)
본의 상하(Z축, 또는 Global 모드에서는 지정한 카메라 Up 방향) 이동 거리를 추적하여 메인 스트로크 데이터를 추출하는 특화 기능 섹션입니다.
* **Stroke 활성화** `[v5.0.0: 기본값 ON으로 변경]`: 본의 이동 데이터를 메인 `.funscript` 파일로 내보냅니다.
* **방향 전환**: 스트로크의 업/다운 방향을 반전합니다.
* **기본 가동 막대 길이 / 배율 (Multiplier)**: 상하 이동 거리를 0~100% 매핑으로 변환할 기준 길이 및 가중치 배율입니다.
* **가동 막대 동적 변형 사용**: 본이 가동 범위를 벗어날 경우 실시간으로 극점을 재계산하여 끊김 없는 유연한 매핑을 지원합니다.
* **기립 판별 사용 (Standing Detection)**: 하반신 높낮이 변화를 실시간 추적하여 상태 전환식 비선형 매핑을 처리합니다.
  * `극점 판단 유예 길이 (Grace Count)`: 정점 감지 시 노이즈 판단을 유예하고 추가로 탐색할 프레임 수입니다. (Pitch/Roll 동적 변형과 공용으로 사용됩니다.)
  * `판단 유예 오차 범위 (%)`: 미세한 진동 노이즈로 인해 기립 상태가 오판되는 것을 방지하는 허용 편차입니다.
  * `보정 발동 임계값 (Transition Threshold Percent)`: 큰 폭의 전이 단계에 진입할 때, "어느 수준의 오차부터 강제 정밀 매핑 보정을 수행할 것인가"를 결정하는 수치입니다. Stroke뿐 아니라 Pitch/Roll 동적 변형에도 동일하게 사용됩니다.
    오차를 넘을 정도의 큰 움직임이 funscript에 충분히 담기지 않았다고 판단하면 강제로 공간을 확보합니다.
  * `대퇴골 뼈 지정`: 기립 여부를 정밀 대조하기 위한 대퇴골(Thigh) 본 이름입니다. (자동 필터링 추천 지원)
  * `판단 막대 길이`: 기립 판별 상태 전환의 기준이 되는 영역의 스케일 길이입니다.

> 💡 위 "임계 탐색 및 보정 상세 파라미터" 박스는 Stroke/기립 판별뿐 아니라 Pitch 또는 Roll 동적 변형을 켰을 때도 함께 표시되는 **공용** 설정입니다.

## 💾 실행 및 디버그 (Execution)

UI 패널 최하단에는 두 개의 버튼이 위치해 있습니다.

1. **Funscript 파일 생성 (Export)**:
   * 설정한 옵션과 RDP 압축 알고리즘에 맞춰 고밀도/저용량으로 정제된 최종 `.funscript` 파일들을 지정 경로에 추출합니다.

# vmd2funscript
idea by miku

## 🛠 주요 기능 (Key Features)

### 1. 드래그 앤 드롭 변환 (Drag & Drop Conversion)

* **간편함:** 복잡한 명령어나 설정 없이 `!VMD!.bat`에 VMD 파일을 끌어다 놓기만 하면 즉시 `.funscript`가 생성됩니다.
* **일괄 처리:** 여러 개의 VMD 파일을 한꺼번에 드래그하여 대량 변환이 가능합니다.

### 2. 모델 맞춤형 스켈레톤 추출 (PMX Skeleton Extraction)

* **정밀도 향상:** 표준 체형 기본값을 넘어, 실제 사용하는 모델의 PMX 파일을 `!PMX!.bat`에 드래그하면 모델별 본(Bone) 위치를 실측하여 변환에 반영합니다.
* **자동 적용:** 추출된 `skeleton.json` 파일이 폴더에 있으면, 변환 시 자동으로 인식하여 더 정확한 모션 추적이 가능합니다.

### 3. 옵션 설정 GUI, 애드온과 자동 연동 `[v5.0.0: 전면 개편]`

* **더 이상 콘솔 Q&A가 아닙니다:** `make_bat.py`를 실행하면 이제 **Tkinter 창(GUI)**이 열립니다. 터미널에 하나씩 답을 입력하던 이전 방식은 폐지되었습니다.
* **애드온 옵션을 실시간으로 읽어옴:** 같은 폴더의 `mmd_to_funscript*.py`(Blender 애드온)를 자동으로 찾아 그 안의 옵션 클래스를 코드 구조 분석(AST)만으로 파싱합니다. **Blender 설치가 필요 없습니다.**
* **애드온이 바뀌면 GUI도 자동으로 최신화:** 애드온에 옵션이 추가/삭제/변경돼도 `make_bat.py`를 다시 실행하기만 하면 GUI가 자동으로 그 변경사항을 반영합니다. `make_bat.py` 코드 자체를 손댈 필요가 없습니다.
* **패널처럼 그룹화된 화면:** 출력 / 탐색 및 압축 / 뼈 지정 / 추출 방식 / Pitch / Roll / Twist / Stroke / 기립 판별·동적 변형 등으로 옵션이 자동 분류되어 표시됩니다.
* **입력값 검증 & 설명 툴팁:** 정수/실수 형식, min/max 범위를 제출 시 검사해주고, 각 항목 옆 `?` 버튼으로 옵션 설명을 바로 확인할 수 있습니다.
* **지속성:** 완성한 설정은 그대로 `.bat` 파일에 고정되므로, 매번 GUI를 다시 열 필요 없이 초기 설정값 그대로 반복 실행할 수 있습니다.

### 4. 고성능 파이썬 엔진 (Standalone Engine)

* **블렌더 불필요:** 블렌더나 MMD 관련 애드온 없이 오직 파이썬 환경만으로 독립 실행됩니다.
* **원본 알고리즘 계승:** 별도의 사전 추출 파일 없이, 실행할 때마다 같은 폴더의 `mmd_to_funscript*.py`(Blender 애드온) 원본 코드에서 연산 블록을 그 자리에서 직접 읽어(AST 파싱, sha256 해시로 추출 결과 확인) 사용합니다. 애드온 파일을 교체하면 다음 실행부터 새 버전의 로직이 자동으로 반영됩니다.
* **음악 싱크 최적화:** VMD 0프레임을 정확히 0ms로 계산하여, 기존 블렌더 임포트 시 발생하던 타이밍 밀림 현상이 없습니다.

### 💡 활용 팁 (Pro-tips)

* **회전 전용 모드:** 스트로크(상하 이동) 없이 회전 값만 추출하고 싶다면 `make_bat.py` GUI에서 Stroke 활성화를 끄거나, 명령줄에서 `--no-export-stroke` 옵션을 사용하세요.
* **디버그 모드:** 변환 결과가 의도와 다르다면 `--debug` 옵션을 통해 funscript 대신 연산 디버그 로그(`.debug.txt`)를 생성하여 원인을 파악할 수 있습니다.
* **옵션을 다시 바꾸고 싶다면:** `make_bat.py`를 다시 실행하면 GUI가 새로 열리며, 이때 생성되는 `.bat` 파일이 이전 설정을 덮어씁니다.
